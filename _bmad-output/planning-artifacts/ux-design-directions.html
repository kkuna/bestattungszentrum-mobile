Primary Direction · Implementation Reference

Quiet OTTO Native

This is the closest adaptation of the live OTTO app screenshots, toned down for the funeral-industry context. It keeps the native-feeling gray canvas, large search pill, white rounded modules, two-column category grid, red active tab indicator, and horizontal content rhythm, but removes retail noise, coupon energy, price pressure, and the purple assistant treatment.

  • Best candidate for the final mobile shell: familiar and modern, but calm enough for sensitive work.
  • Uses OTTO's real in-app rhythm: search at the top, modular content below, stable bottom navigation.
  • Replaces shopping modules with supplier categories, RFQ status cards, verified supplier tiles, and request shortcuts.

Supporting Pattern · Forms

Guided Anfrage

Form-first and completion-oriented. The UI makes the category schema feel like a guided checklist, then closes with a professional receipt.

  • Best for the defining RFQ creation flow.
  • Elevates review-before-submit and email-aware confirmation.
  • Useful as a pattern even if another direction wins overall.

Supporting Pattern · History

Quiet Ledger

A precise, compact direction for request history and timelines. It leans into auditability: clear metadata, stable rows, and fewer decorative surfaces.

  • Best for request history, quote timeline, and account/security screens.
  • Useful when users compare statuses and timestamps repeatedly.
  • Should be softened with brand moments elsewhere to avoid feeling administrative.

Supporting Pattern · Brand & Status

Warm Institutional

More brand-led and ceremonial, using deeper ink, serif titles, warm paper, and restrained red. It feels premium and sober, but risks being slower for dense workflows.

  • Best for onboarding, signup, empty states, and brand moments.
  • Use sparingly inside operational screens to avoid reducing scan speed.
  • Good source for splash and account status screens.

Archived Exploration

Explored, Not Reference

The earlier directions are intentionally removed from the implementation path. They were useful for exploration, but the app should now be built from one primary direction and three supporting pattern libraries.

  • Operational Red informed the red active/action system, but is too app-bar-heavy for the final shell.
  • Supplier Command informed inbox density, but the dark chrome is too strong for the baseline supplier experience.
  • Blue Dispatch Accent remains only as a semantic info/dispatch color idea, not a screen direction.
  • Restrained OTTO Framework was superseded by Quiet OTTO Native after the live app screenshots.
