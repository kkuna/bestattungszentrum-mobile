## What Went Well

- **The code-review loop is the standout.** Every story (1.1–1.7) went through review and every one surfaced genuine correctness/security bugs before "done" — fail-open route gates, logout not clearing local session on backend failure, 403→auth mis-mapping, un-encoded route IDs, AA contrast failures, locale-persistence bugs. This is the practice to protect.
- **Strong typed architectural boundaries:** typed session-storage boundary, normalized API result/problem boundary, and a centralized fail-closed account-access decision. A coherent vertical foundation for Epics 2–4.
- **Consistent quality discipline:** all gates enforced per story; per-story Argent verification (Android).

## Challenges / Growth Areas

- **Recurring bug class — auth/session fail-closed semantics.** The *same shape* of finding recurred across 1.4, 1.5, and 1.6 ("doesn't fail closed", "doesn't clear local state on failure", "recoverable error signs user out"). Review caught them, but the repetition drove rework. **A pre-review self-checklist for auth/session/gate code would cut the rework drag** (the Project Lead's noted pain point) without weakening the safety net.
- **iOS runtime verification blocked since Story 1.1.** Dev-client redbox `Failed to call into JavaScript module method RCTEventEmitter.receiveEvent(). Module has not been registered as callable`. Build/install succeed; JS redboxes before render. Whole epic was verified Android-only. iOS risk has accumulated.
- **Backend mobile contract gap.** `/api/mobile/auth/login` and `/api/mobile/auth/me` omit `accountStatus`/`userStatus`, so a real login fails closed to "Status unbekannt" and cannot reach an active workspace. Also, `verificationStatus: FAILED` was initially mis-modeled as an `accountStatus` value (1.6 finding). The 1.7 review verification required a temporary active-session fixture to see active tabs at all.

## Key Takeaways

1. The per-story code-review loop catches real security/correctness bugs — keep it, and front-load a fail-closed self-checklist to cut rework.
2. Auth/session/gate code is the project's highest-risk surface; treat fail-closed semantics and local-state cleanup as defaults, not review catches.
3. Verify on the platform you ship: the iOS blocker and the backend contract gap both limited real verification this epic — resolve iOS before Epic 2, coordinate the backend contract in parallel.
